## 1. 测试目标

本文档用于系统性地测试校园点餐系统**学生端、商家端、管理员端**的全部核心功能，验证系统是否满足以下要求：

- 点餐、购物车、下单、支付、订单处理的完整业务流程
- 商家入驻申请、管理员审核、账号生成的管理流程
- 菜品管理（增删改查、上下架）的正确性
- 销售统计分析的数据准确性
- 消息聊天功能的可用性
- 管理员系统管控的有效性
- 三个角色之间的权限隔离
- 异常输入和越权操作的拦截能力

## 3. 测试前准备

### 3.1 启动服务

**步骤1：启动 MySQL**

确保 MySQL 服务已启动，端口为 3306。

```bash
# Windows 检查
net start | findstr MySQL

# macOS 检查
brew services list | grep mysql
```

**步骤2：创建数据库**

```sql
CREATE DATABASE IF NOT EXISTS campus_dining 
  DEFAULT CHARACTER SET utf8mb4 
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

**步骤3：配置并启动后端**

检查 `src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_dining?...
    username: root
    password: 你的密码   # ← 改成你自己的
```

启动后端：
```bash
cd campus-dining
mvn spring-boot:run
```

看到以下输出表示启动成功：
```
校园自助点餐系统启动成功！
Knife4j 文档: http://localhost:8080/doc.html
```

**步骤4：启动前端（Vue工程）**

```bash
cd front
npm install        # 仅第一次
npm run dev
```

看到以下输出表示启动成功：
```
VITE v8.0.x  ready in xxx ms
➜  Local:   http://localhost:5173/
```
